如何在基于hexo + github 的博客文章中插入图片🖻

前言

我们知道，基于 Hexo+GitHub Pages 可实现远程的静态博客功能，在实际使用过程中，我们如果在某一篇博客内添加了图片，往往会由于图片的引用路径和远程仓库中图片路径不一致，导致在本地正常显示的图片部署到 github 仓库🐹后出现图片无效的问题。我们可以通过以下方法解决以上问题。

1、新建img文件夹📁

在站点目录下的source目录下新建img文件夹：

2、设置Typora的图片插入🔌

找到Typora的【偏好设置】 –> 【图片插入】，选择【复制图片到./${filename}.assets文件夹】

并勾选以下三个功能选项：

3、设置当插入本地图片时复制到文件夹📁

找到Typora的【编辑】选项卡 –>【图片工具】–>【当插入本地图片时】–>【复制到文件夹…】：

选择在source目录下新建的img文件夹：

点击确认后我们可以发现文件头出现typora-copy-images-to: ..\img 的字样，表示我们配置成功：

4、设置图片根目录📇

找到Typora的【编辑】选项卡 –>【图片工具】–>【设置图片根目录】

选择站点目录下的source文件夹，也就是img文件夹的上级：

点击确认后我们可以发现文件头出现typora-root-url: .. 的字样，表示我们配置成功
